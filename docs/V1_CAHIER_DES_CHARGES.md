# Cahier des charges — Version 1

## 1. Identification du projet

**Projet :** Portfolio Olivier Polynice  
**Version :** V1 — Première page d’accueil complète  
**Branche Git :** `feature/v1-homepage`  
**Technologie principale :** React avec Vite  
**Auteur :** Olivier Polynice  

## 2. Objectif principal de la V1

L’objectif principal de la V1 est de créer une première page d’accueil complète, moderne, responsive et cohérente.

Cette version doit permettre d’obtenir un portfolio déjà présentable à un recruteur, même si certaines fonctionnalités avancées seront ajoutées dans les versions suivantes.

La V1 doit mettre l’accent sur :

- la lisibilité ;
- la crédibilité professionnelle ;
- la qualité visuelle ;
- la simplicité de navigation ;
- la cohérence des informations ;
- la compatibilité mobile ;
- la qualité du code React.

## 3. Résultat attendu à la fin de la V1

À la fin de la V1, le portfolio devra contenir les sections suivantes :

- barre de navigation ;
- section d’accueil ;
- section À propos ;
- section Compétences ;
- section Projets ;
- section Parcours ;
- section CV ;
- section Contact ;
- pied de page.

Le site devra également comporter :

- un design sombre professionnel ;
- une navigation fluide entre les sections ;
- des boutons fonctionnels ;
- un CV téléchargeable ;
- des liens GitHub et LinkedIn ;
- un affichage responsive ;
- des données centralisées dans les fichiers JavaScript ;
- des composants React réutilisables ;
- une documentation V1 ;
- des captures de validation ;
- une branche Git dédiée ;
- une Pull Request vers `main`.

## 4. Structure fonctionnelle de la page

La page d’accueil suivra cet ordre :

1. Navbar ;
2. Hero ;
3. À propos ;
4. Compétences ;
5. Projets ;
6. Parcours ;
7. CV ;
8. Contact ;
9. Footer.

## 5. Critères de qualité

La V1 devra respecter les critères suivants :

- aucune erreur importante dans la console ;
- aucune erreur d’importation ;
- compilation réussie avec `npm run build` ;
- navigation fonctionnelle ;
- composants React séparés ;
- données séparées de l’interface ;
- affichage compatible mobile, tablette et ordinateur ;
- contraste suffisant entre le texte et l’arrière-plan ;
- boutons et liens accessibles au clavier ;
- images avec attributs `alt` adaptés.
